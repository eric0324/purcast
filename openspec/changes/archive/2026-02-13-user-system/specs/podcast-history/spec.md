## ADDED Requirements

### Requirement: Podcast history page
The system SHALL provide a page listing all podcasts generated by the current user.

#### Scenario: Display podcast list
- **WHEN** a user visits the history page
- **THEN** the system SHALL display podcasts sorted by creation date (newest first), showing title, date, duration, and status

#### Scenario: Empty history
- **WHEN** a user has no generated podcasts
- **THEN** the page SHALL display an empty state with a CTA to create their first podcast

#### Scenario: Pagination
- **WHEN** the user has more than 10 podcasts
- **THEN** the system SHALL load more items via cursor-based pagination (infinite scroll or "Load more" button)

### Requirement: Podcast status display
The system SHALL display the current generation status for each podcast.

#### Scenario: Pending podcast
- **WHEN** a podcast has status 'pending' or 'generating_script' or 'generating_audio'
- **THEN** it SHALL display a spinner with the current step text

#### Scenario: Completed podcast
- **WHEN** a podcast has status 'completed'
- **THEN** it SHALL display the duration and a play button

#### Scenario: Failed podcast
- **WHEN** a podcast has status 'failed'
- **THEN** it SHALL display an error indicator with option to retry

### Requirement: Podcast detail view
The system SHALL allow users to view details of a generated podcast.

#### Scenario: View podcast detail
- **WHEN** a user clicks on a completed podcast
- **THEN** the system SHALL display the title, source content preview, audio player, duration, and creation date

#### Scenario: Download podcast
- **WHEN** a user clicks "下載" on a completed podcast
- **THEN** the browser SHALL download the MP3 file
